# Deep Research 核心架构与研究思路

## 概述

Deep Research 是 RequirementAgent 项目中的一个自动化深度研究功能模块。它能够接收用户的研究问题，自动完成从查询分析、研究规划、多源并行搜索、内容综合到最终生成专业研究报告（PDF/Word）的全流程。整个流程基于 **LangGraph 状态图**驱动，由多个专职 Agent 协作完成。

---

## 整体研究流程

Deep Research 的核心 pipeline 由 5 个串行阶段组成：

```
用户查询 → 查询分析 → 研究规划 → 并行搜索与综合 → 报告撰写 → 报告生成（PDF/Word）
```

### 阶段一：查询分析（Query Analyzer）

接收用户的原始查询，对其进行深层理解：
- 识别用户意图和研究领域（如科技、人文、医学、商业等）
- 提取关键概念
- 检测用户语言（支持中英文等多语言）
- 将原始查询扩展为更精确的研究问题

这一步确保后续所有环节都基于对用户需求的准确理解。

### 阶段二：研究规划（Planner）

基于扩展后的查询，自动生成 8-12 个结构化研究任务：
- 每个任务包含具体研究方向、优先级和类别
- 类别覆盖：概述、历史背景、技术分析、案例研究、对比分析、挑战与展望等
- 确保研究的全面性和系统性

### 阶段三：并行搜索与综合（Parallel Searcher）

这是研究的核心执行阶段，采用 **4 个子搜索 Agent 并行工作**：
- **通用网络搜索**：通过搜索引擎获取广泛信息
- **学术搜索**：从 arXiv、IEEE 等学术源获取专业文献
- **文档搜索**：从官方文档站点获取权威资料
- **图片搜索**：获取相关图表和可视化素材

每个子任务完成搜索后，由 LLM 对结果进行综合提炼，确保信息质量。若搜索失败，系统会回退到 LLM 自身知识作为补充。

### 阶段四：报告撰写（Writer）

将所有搜索结果和综合信息组织成完整的研究报告：
- 根据研究领域自动选择合适的报告结构（不同领域有不同的章节模板）
- 生成包含执行摘要、引言、核心分析、挑战与展望、结论、参考文献等完整章节
- 自动生成 Mermaid 图表辅助可视化
- 报告语言与用户查询语言一致
- 对于大规模内容（>30k 字符），采用分章节撰写策略

### 阶段五：报告生成（PDF Generator）

将 Markdown 格式的报告转化为专业排版的 PDF 和 Word 文档：
- 渲染 Mermaid 图表为图片嵌入
- 处理 LaTeX 数学公式
- 下载并嵌入网络图片
- 解析和渲染 Markdown 表格
- 支持中文字体排版

## 架构设计理念

### 1. Agent 协作模式

采用 **流水线式多 Agent 协作**，每个 Agent 专注于单一职责：

| Agent | 职责 |
|-------|------|
| QueryAnalyzerAgent | 理解用户意图，扩展查询 |
| PlannerAgent | 制定研究计划 |
| SearcherAgent | 协调并行搜索 |
| SubSearchAgent (×4) | 执行具体搜索与综合 |
| WriterAgent | 撰写完整报告 |
| PDFReportGenerator | 生成最终文档 |

### 2. 状态驱动的工作流

基于 LangGraph 的 `ResearchState` 在各节点间传递和累积研究数据，保证信息流的完整性和可追溯性。

### 3. 多源信息融合

通过并行搜索多种信息源（网络、学术、文档），再由 LLM 进行综合，确保研究报告的信息广度和深度。

### 4. 领域自适应

系统根据检测到的研究领域（科技、人文、医学、商业、法律等）自动调整报告结构和写作风格，使输出更具专业性。

### 5. 容错与回退

搜索失败时自动回退到 LLM 知识；报告过短时自动触发分章节重写；解析失败时使用预设默认值。

---

## 系统交互方式

Deep Research 支持两种触发方式：
1. **直接 API 调用**：通过 REST API 提交研究任务，异步获取结果
2. **对话触发**：在聊天中自动识别深度研究意图，无缝启动研究流程

研究过程为异步执行，用户可随时查询进度和下载最终报告。
